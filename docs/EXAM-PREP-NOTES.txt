DOMAIN 1: HIGH AVAILABILITY (34% of exam)

KEY CONCEPTS:
- Multi-AZ deployment: Primary + standby across AZs
- No single point of failure
- Load balancing: ALB, NLB
- Auto Scaling: Dynamic capacity
- Health checks: Remove unhealthy instances

SERVICES:
- Route 53: DNS failover
- ALB/NLB: Load balancing
- ASG: Auto-scaling
- RDS Multi-AZ: Database failover
- Read Replicas: Scale reads
- CloudWatch: Monitoring
- ElastiCache: Cache HA

UPTIME TARGETS:
- 99.9% = 8.7 hours/year downtime
- 99.99% = 52 minutes/year downtime
- 99.999% = 5 minutes/year downtime

FAILOVER PROCESS:
1. Health check detects failure
2. Route 53 updates DNS
3. Traffic redirects
4. Standby promoted (if applicable)
5. New standby created

EXAM SCENARIOS:
- Design 99.99% uptime: Multi-AZ + ASG + Multi-AZ RDS
- Eliminate single point: Add redundancy at each layer
- Handle traffic spike: ASG + load balancer
- Database HA: Multi-AZ or read replicas

DOMAIN 2: DESIGN FOR SECURITY (30% of exam)

KEY CONCEPTS:
- IAM: Access control
- VPC: Network isolation
- Encryption: Data protection
- Audit: CloudTrail logging
- Principle of Least Privilege (PoLP)

IAM:
- Users: Individual accounts
- Groups: Collections of users
- Roles: Temporary credentials
- Policies: Permission documents
- MFA: Multi-factor authentication

VPC SECURITY:
- Security Groups: Stateful, instance-level
- NACLs: Stateless, subnet-level
- Public subnets: Internet access
- Private subnets: No internet
- VPC Endpoints: Private AWS access

ENCRYPTION:
- At rest: KMS, S3 SSE, RDS encryption
- In transit: TLS/HTTPS
- ACM: Free SSL certificates
- Secrets Manager: Store sensitive data

AUDITING:
- CloudTrail: API audit logs
- CloudWatch Logs: Application logs
- AWS Config: Configuration tracking
- GuardDuty: Threat detection
- VPC Flow Logs: Network monitoring

SHARED RESPONSIBILITY:
- AWS: Infrastructure, facilities, services
- Customer: IAM, data, application, access
